# Учим и играем

Развивающее веб-приложение (PWA) для Тимура и Самиры. Работает на планшете офлайн после установки на домашний экран, без Play Store.

## Установка на планшет

1. Открыть ссылку `https://klero7.github.io/first/` в Chrome на планшете.
2. Открыть меню Chrome (⋮) → «Добавить на главный экран».
3. Дальше приложение открывается как обычная иконка, работает без интернета.

## Разработка

```bash
npm install
npm run dev       # локальный сервер разработки
npm run build     # сборка в dist/
npm run preview   # предпросмотр собранной версии
```

Деплой на GitHub Pages происходит автоматически (`.github/workflows/deploy.yml`) при пуше в ветку `main`.

## Как добавить или изменить задания

Всё содержимое заданий лежит в папке `content/` в виде markdown-файлов — трогать код в `src/` для этого не нужно.

Структура: `content/<ребёнок>/<категория>/<файл>.md`, где `<ребёнок>` — `timur` или `samira`.

Пример файла (`content/timur/math/addition-within-10-rockets.md`):

```markdown
---
id: timur-addition-rockets-01
child: timur
category: math
type: addition
title: "Сложение с ракетами"
theme: space
difficulty: 1
emoji: "🚀"
roundSize: 5
questions:
  - id: q1
    prompt: "2 + 3 = ?"
    display: "count-visual"
    visualAsset: rocket
    addends: [2, 3]
    choices: [{"label":"4","correct":false},{"label":"5","correct":true},{"label":"6","correct":false}]
    speak: "Два плюс три?"
---
Необязательный текст-подсказка, отображается перед раундом.
```

### Поля файла

- `id` — уникальный идентификатор набора заданий.
- `child` — `timur` или `samira`.
- `category` — для группировки на главном экране (`counting`, `math`, `letters`, `reading`, `numbers`, `matching`).
- `theme` — `space` (космос, для Тимура) или `cosmetics` (косметика/доктор, для Самиры).
- `difficulty` — 1-3, влияет на порядок сортировки на главном экране.
- `roundSize` — сколько вопросов показывается за один раз (для Самиры держим 5 — короткие раунды).
- `questions` — список вопросов, из которых случайно выбирается `roundSize` штук при каждом прохождении. Можно добавлять сколько угодно вопросов в файл — раунд всё равно останется коротким.

### Типы вопросов (`display`)

| `display` | Что показывает | Обязательные поля |
|---|---|---|
| `multiple-choice` | Текст/картинка-эмодзи + варианты ответа | `choices` |
| `count-visual` | Сложение/вычитание с картинками (повторяется эмодзи) | `visualAsset`, `addends` (например `[5, -2]` — это 5 минус 2), `choices` |
| `tap-count` | Считаем предметы, нажимая на каждый | `visualAsset`, `count`, `choices` |
| `number-line` | «Какое число дальше?» | `sequenceStart`, `choices` |
| `letter-card` | Большая буква/цифра + похожие варианты | `visualAsset` (сама буква/цифра), `choices` |
| `match-pairs` | Соедини пары (нажатием) | `pairs: [{left, right}]` |

Для `choices` ровно один вариант должен быть `correct: true`. Для `visualAsset` в `count-visual`/`tap-count` используется условный ключ картинки (`rocket`, `planet`, `lipstick`, `syringe` и т.д. — полный список в `src/games/assets.ts`), а в `letter-card`/`multiple-choice` — просто сам символ/эмодзи для показа.

### Как обновить контент

1. Отредактировать/добавить `.md` файл в `content/` (можно попросить Claude сделать это прямо в чате).
2. Запустить `npm run build`, чтобы проверить, что нет ошибок (в консоли dev-режима также видны предупреждения о неверной разметке файла).
3. Закоммитить и запушить в `main` — GitHub Actions автоматически соберёт и задеплоит новую версию.
4. На планшете, когда приложение уже установлено и открыто, появится баннер «Доступно новое задание! Обновить» — по нажатию подтягивается новая версия.

## Технологии

- React + TypeScript + Vite
- `vite-plugin-pwa` (офлайн-кэш, манифест, установка на домашний экран)
- Web Speech API (`ru-RU`) для озвучки — если голос недоступен на устройстве, приложение продолжает работать без звука
- Прогресс и звёзды хранятся в `localStorage`, отдельно для каждого ребёнка
